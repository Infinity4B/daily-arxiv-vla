SpecPrune-VLA: Accelerating Vision-Language-Action Models via Action-Aware Self-Speculative Pruning

2025-09-06 · 原文链接 · 幻觉翻译 · 2509.05614

论文研究单位

未在提供的文本中明确提及。

论文概述

本文提出了一种名为SpecPrune-VLA的免训练剪枝方法，用于加速视觉-语言-动作（VLA）模型的推理。该方法通过结合当前动作生成的局部信息和先前动作的全局信息来选择token，实现两级剪枝和启发式控制。实验表明，在LIBERO模拟基准测试中，相比高性能模型OpenVLA-OFT，该方法在NVIDIA A800和RTX 3090 GPU上分别实现了1.46倍和1.57倍的平均加速，且任务成功率损失可忽略不计。

论文核心贡献点

  1. 提出了一种新见解：连续动作之间的信息高度相似，应结合当前动作的局部信息和先前动作的全局信息进行token选择。
  2. 设计了SpecPrune-VLA框架，包含三个技术：动作级静态token剪枝、层级动态token剪枝和轻量级动作感知控制器。
  3. 实现了在不同硬件平台上的高效推理加速，且保持任务成功率。

论文方法描述

  1. 动作级静态剪枝：利用先前动作的全局注意力信息选择top-K全局重要token，通过基于速度的帧比较和自推测token选择补充动态token和任务相关token，融合局部和全局选择后剪枝50-70%的视觉token。
  2. 层级动态剪枝：通过动态更新token的重要性分数并在不同层重新评估token重要性，引入基于排名的权重和层级置信度分数计算最终分数。
  3. 轻量级动作感知控制器：根据末端执行器速度将动作分为粗粒度（如大位移）和细粒度（如抓取），动态调整剪枝策略，以平衡速度和精度。

论文使用数据集和训练资源

  • 数据集：LIBERO模拟基准，包含四个任务套件（LIBERO-Spatial、LIBERO-Object、LIBERO-Goal、LIBERO-Long），每个套件10个任务。
  • 训练资源：未提及训练细节，方法为免训练，基于OpenVLA-OFT实现，实验在NVIDIA A800-80GB和RTX 3090 GPU进行。

论文使用的评估环境和评估指标

  • 评估环境：Linux工作站，使用NVIDIA A800-80GB和RTX 3090 GPU，模拟环境为Franka Emika Panda机械臂。
  • 评估指标：

- 任务成功率（Success Rate, SR）：衡量任务完成的百分比。

- 延迟（Latency）：从接收输入到生成动作的时间，单位毫秒。

- 加速比（Speedup）：相对于基线模型的延迟减少倍数。

- FLOPs：计算量，用于量化剪枝效果。
