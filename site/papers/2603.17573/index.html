Card 02 论文概述

论文概述

  • 论文旨在解决具身视觉-语言-动作（VLA）模型推理速度慢的问题，这是该领域实现实时控制的主要瓶颈。
  • 研究分析了两种推测解码（Speculative Decoding， SD）方法：基于起草模型的方法和基于检索的方法，并发现二者在VLA模型中各有优劣，应混合使用以达到更优性能。
  • 论文提出了一个名为 HeiSD 的端到端混合推测解码框架，通过优化检索式SD并设计运动学感知的边界确定策略，在保持高任务成功率的同时实现显著加速。
Card 03 核心贡献

核心贡献

  • 通过对机器人轨迹模式的深入分析，得出关键见解：在VLA模型的推理过程中混合使用基于起草模型的SD和基于检索的SD能实现更优的性能平衡。
  • 提出了检索式SD的优化方法，包括自适应验证跳过机制和序列级宽松接受策略，以解决检索草稿质量低和持久错误的问题。
  • 设计了一种基于运动学的融合指标，用于在多步推理过程中自动、动态地确定两种SD方法的使用边界。
  • 构建并开源了完整的 HeiSD 框架，在模拟基准和真实世界场景中实现了高达 2.45× 和 2.41× 的加速，同时保持了高任务成功率。
Card 04 方法描述

方法描述

  • HeiSD 框架的核心是混合推测解码策略，根据轨迹段的运动学特征自动选择使用检索式SD或基于起草模型的SD。
  • 对于检索式SD的优化：提出了自适应验证跳过机制，根据历史轨迹特征相似性模式选择性跳过验证过程；提出了序列级宽松接受策略，将具有运动学关联的token分组为序列进行验证，允许整体偏差在特定范围内的序列被接受，从而减少持久错误并提高草稿多样性。
  • 为确定混合边界：分析了轨迹段的曲率半径和累积位移等运动学特征，提出了一个融合这两个特征的归一化指标，用于在每一步推理中自动判断应采用哪种SD方法。
Card 05 数据集与资源

数据集与资源

  • 主要评估使用 LIBERO 仿真基准数据集，包含 LIBERO-Goal、LIBERO-Object、LIBERO-Spatial 和 LIBERO-Long 四个任务套件。
  • 基础VLA模型采用 OpenVLA；用于基于起草模型的SD的草稿模型是一个单层 LLaMA 块。
  • 实验硬件平台包括 NVIDIA A100 GPU 和 Intel Xeon Silver 4410T CPU；检索数据库使用 Qdrant 向量数据库构建，部署在CPU内存中以降低GPU显存开销。
Card 06 评估与结果

评估与结果

  • 评估在 LIBERO 仿真基准和真实机器人桌面操作环境中进行，主要评估指标为任务成功率（SR）和推理加速比。
  • 在仿真实验中，HeiSD 相比标准自回归推理实现了 1.79× 至 2.45× 的加速；相比最优基线方法 SpecVLA，实现了 1.51× 至 2.22× 的进一步加速，且成功率更高。
  • 在真实世界实验中，HeiSD 实现了 2.06× 至 2.41× 的加速，同时维持了与基线方法相当甚至更高的任务成功率。
  • 实验结果验证了混合使用两种SD方法的有效性，以及所提出的优化策略在加速推理的同时保持任务精度的能力。
