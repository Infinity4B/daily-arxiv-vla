Card 02 论文概述

论文概述

  • 提出 Metis，一个端到端的世界-动作模型（WAM）框架，用于自动驾驶和城市导航
  • 解决现有 WAM 的两大痛点：因未来观测预测导致的推理延迟，以及视频生成与动作建模紧密耦合引起的表示不匹配和泛化能力下降
  • 通过解耦视频生成与动作预测，实现在不牺牲性能的前提下大幅提升推理效率
Card 03 核心贡献

核心贡献

  • 设计 Metis 框架，将动作预测模型与视频生成模型解耦，各自保持分布特性，提升泛化能力
  • 提出 非对称注意力掩码，强制视频token可关注动作token，但动作token不可关注未来视频token，实现训练时联合优化、推理时无需显式生成视频，保证训练-推理一致性
  • 在 NAVSIM-v2（navhard/navtest）和 CityWalker 基准上取得最优性能，并在真实机器人上完成零样本部署验证
Card 04 方法描述

方法描述

  • 采用 Mixture-of-Transformers (MoT) 架构，包含视频生成专家（VGE，基于 Wan2.2-5B）和动作专家（AE，较小扩散Transformer，隐藏维度1024）
  • 引入 非对称注意力掩码：动作token仅关注当前观测token，未来视频token可关注当前观测和所有未来动作token，实现信息单向流动
  • 使用 流匹配（flow matching） 目标联合训练动作预测和未来视频生成，加权系数λ=1
  • 推理时仅通过动作专家从当前观测直接预测动作轨迹，无需视频生成，大幅降低延迟
Card 05 数据集与资源

数据集与资源

  • NAVSIM-v2：navtrain（1192场景）训练，navhard（244真实+4164合成）和navtest（12146场景）评估
  • CityWalker：15小时纽约城市遥操作数据，6小时微调，9小时测试
  • 模型总参数量约 6B（VGE 5B，AE 1B）
  • 训练硬件：8 × NVIDIA H200 GPU，navsim训练60 epochs，CityWalker训练30 epochs，batch size 64
Card 06 评估与结果

评估与结果

  • NAVSIM-v2 navhard：Stage 2 EPDMS 达 32.2，全面超越传统E2E、VLA和WAM方法，尤其在DAC和DDC上提升显著
  • NAVSIM-v2 navtest：EPDMS 达 89.5（单模型），最佳变体 90.3（best-of-6），多个子指标达到或接近最优
  • CityWalker：MAOE均值 11.2，整体 7.6，在转向、路口、绕行等场景中均优于 GNM、ViNT、NoMaD 等基线
  • 零样本真实机器人（Unitree Go2）在室内外环境中验证了泛化能力和实用性
