Reflex: Real-Time VLA Control through Streaming Inference

2026-07-16 · 原文 · 翻译 · 2607.14695

Flow Matching 视觉-语言-动作（VLA）模型因迭代去噪过程存在阻塞执行问题，导致机器人控制需求（50–100Hz）与推理延迟（100–200ms）之间存在数量级差距。 标准 KV-Caching 在 Flow Matching 模型中失效，因为全局时间步长注入使缓存特征在每个去噪步骤中失效，迫使研究者在缓慢的 O(N²) 重计算与数学上不正确的缓存复用之间权衡。 本文提出 Reflex，一个实时流式推理框架，利用时间步长不变性属性（感知编码器与去噪循环函数独立），通过分片注意力、自适应归一化和异步流水线实现 50Hz 稳定流式控制。

Card 03 核心贡献

核心贡献

  • 提出从加速单个推理步骤转向基于流式的异步执行这一根本性转变，以解决实时 VLA 控制中的系统阻塞问题。
  • 设计 Reflex 异步服务架构，包含结构感知缓存（Partitioned Attention）实现 O(1) 增量更新，以及 AdaRMSNorm 确保无限时域混合精度部署的数值稳定性。
  • 在 LIBERO 和 Kinetix 基准上验证，Reflex 将系统反应延迟降低最多 54%，实现 0% 停顿率，并在动态物理任务上显著提升任务成功率。
Card 04 方法描述

方法描述

  • 分片注意力（Partitioned Attention）：将模型上下文分为静态前缀（固定语言指令）、滑动窗口（最近 N 帧视觉观测）和动态后缀（流状态与时间步嵌入）。利用感知编码器的时间步长不变性，仅动态后缀需要重计算，实现 O(1) 缓存更新且输出与全批量注意力完全等价（MSE=0.00）。
  • AdaRMSNorm：精度感知的归一化层，将 RMS 统计量计算严格置于 FP32 以防止 BFloat16 下溢，门控 MLP 保持在 BFloat16 以节省带宽，通过基于机器人状态的自适应增益 γ(c) 稳定高方差流初始化。
  • 异步流水线：将视觉编码（Thread A）与动作生成（Thread B）解耦到独立线程并行执行。采用未来条件状态预测（Future-Conditional State Prediction）以最后 commanded action 近似未来机器人状态，补偿异步延迟；通过自适应重叠调度（Adaptive Overlap Scheduling）动态调整预计算重叠步数 K。
  • 系统优化：实现算子融合（QKV 投影融合、SwiGLU 门控融合）减少 50% kernel launch 开销；采用静态 Ring Buffer 和手动缓存合并（Manual Cache Merging）消除动态内存分配与 torch.cat 开销，确保确定性延迟。
Card 05 数据集与资源

数据集与资源

  • 基础模型：Pi0.5（2.3B 参数：PaliGemma 视觉语言 backbone 2B + action expert decoder 300M）及更大规模的 Pi0（3.1B 参数）。
  • 基准数据集：LIBERO（包含 Spatial、Object、Goal、Long 四类操作任务）与 Kinetix（强调高频物理交互的动态控制任务）。
  • 硬件资源：NVIDIA RTX 4090/3090 用于推理延迟测试；AgileX PiPer 机器人用于真实世界部署验证。
Card 06 评估与结果

评估与结果

  • 评估指标：推理延迟（Inference Latency）、反应延迟（Reaction Latency）、停顿率（Stall Rate）、峰值内存（Peak Memory）、任务成功率（Success Rate）。
  • 系统效率：在 LIBERO 上使用 Pi0.5 实现 2.58× 推理加速（135.2ms → 52.4ms），Pi0 上达到 2.73×；峰值 VRAM 降低 27%（LIBERO）和 24%（Kinetix）；分片注意力与全批量注意力的输出 MSE 严格为 0.00。
  • 控制响应性：反应延迟最高降低 54%（如 LIBERO-Long 从 168.8ms 降至 84.2ms），实现 0% 停顿率（对比同步基线的 100%），稳定支持 50Hz 控制。
  • 任务性能：在 LIBERO 上保持性能持平（LIBERO-Long 提升 +3.6%），在 Kinetix 动态任务上成功率提升显著（Pi0.5 提升 +7.4%，Pi0 提升 +6.7%）。
  • 真实机器人部署：在 AgileX PiPer 上，Reflex 相比 Sync 基线在 Pick-Place、Articulated、Dynamic Recovery 任务上成功率分别提升 +11pp、+14pp、+17pp，反应延迟维持在 101–110ms。
  • 消融实验：AdaRMSNorm 将 BFloat16 稳定步数从 120–220 步提升至 >2000 步；算子融合使 action expert 前向延迟降低 18%；上下文窗口 K=10 在速度与精度间达到最优平衡。
