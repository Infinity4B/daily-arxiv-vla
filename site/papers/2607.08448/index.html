Card 03 核心贡献

核心贡献

  • 提出一种记忆增强的代理框架，将冻结VLA作为固定原语（vla_act），与少量分析原语组合，通过规划器学习调度，而非扩展原语库，从而将VLA从局部接触控制扩展到长时域、扰动操控
  • 通过经验分析证明，当规划器学习每个原语的操作范围时，小型固定原语库是足够的：规划器分阶段调用可重试VLA，分析原语负责非接触结构，稀疏VLA调用与确定性控制交替
  • 在多个基准测试上取得显著提升：在 LIBERO-Pro 上比最强相关基线 RATS 提高 38.6 个百分点，在 RoboCasa365 上提高 25.4 个百分点，在 RoboTwin C2R 上达到 58.4% 成功率，同时保持标准 LIBERO 的竞争性性能
Card 04 方法描述

方法描述

  • 采用 REPL 风格的代理执行循环，规划器（Codex 或 Claude Code）接收RGB-D观察和任务描述，通过JSON调用固定原语库中的原语，原语执行直到内部后置条件满足
  • 原语库包含两类：分析原语（如 move_to、rotate_wrist、set_gripper 等）用于非接触结构，以及 VLA原语（vla_act）用于接触丰富交互；原语库固定，部署时不允许新增
  • 引入两个记忆模块：任务特定记忆 存储成功原语轨迹，用于少样本重新接地；全局记忆 存储通用成功规则和失败模型，避免重复错误
  • 通过探索引导阶段（在参考种子任务上自主探索，生成可重用的记忆）和部署评估阶段（严格执行，禁用重置，利用记忆进行泛化）实现
Card 05 数据集与资源

数据集与资源

  • 使用的数据集：LIBERO（标准桌面操控）、LIBERO-Pro（带指令重定向和位置交换扰动）、RoboCasa365（家庭厨房操控）、RoboTwin C2R（双臂清洁到随机化转移）
  • 模型规模：冻结VLA包括 RLinf（基于 π0.5 的 SFT 检查点）、RLDX-1（RoboCasa 检查点）、LingBot-VLA（后训练检查点）；规划器使用 Codex 和 Claude Code 大语言模型
  • 训练资源：论文未明确说明计算资源，但VLA为预训练冻结模型，代理执行在API调用和模拟器之间进行
Card 06 评估与结果

评估与结果

  • 评估环境：在 LIBERO、LIBERO-Pro、RoboCasa365、RoboTwin C2R 四个基准上评测，每个任务使用多个种子（如100次试验）和严格的部署协议
  • 主要评估指标：任务成功率（%）
  • 关键实验结果：

- 在标准 LIBERO 上，Harness VLA (CC) 达到 96.0% 总体成功率，与冻结VLA（95.3%）相当

- 在 LIBERO-Pro 上，Harness VLA (CC) 以 82.4% 总体成功率大幅超越 RATS（43.8%）和直接VLA（50.0%）

- 在 RoboCasa365 上，Harness VLA (Codex) 达到 55.4%，比 RLDX-1（30.0%）提升 25.4 个百分点

- 在零样本设定下，Harness VLA 仍展现较强的迁移能力，尽管性能低于少样本

- 机制分析表明：规划器级语义重新接地恢复任务条件行为，分阶段VLA调用提高可靠性，分析原语有效隔离非接触执行
