Card 02 论文概述

论文概述

  • 针对视觉-语言-动作模型（VLA）全量推理计算成本高、难以满足低延迟高频闭环控制需求的问题，提出异步语义-动作解耦框架
  • 框架将低频语义理解模块与高频动作生成模块分离，沿现有VLA的内部语义-动作接口进行解耦，无需重新设计视觉-语言骨干网络或引入外部规划器
  • 解决了在语义缓存可能过时的情况下，如何维持稳定的高频状态反馈控制的核心问题
Card 03 核心贡献

核心贡献

  • 提出一种低侵入性的异步语义-动作解耦框架，允许重量级理解模块低频运行而动作模块以控制频率高频运行
  • 引入历史动作条件化和时间错位训练机制，以缓解过时语义条件与当前执行状态之间的时间不匹配，提升反馈控制鲁棒性
  • 在 $\pi_{0.5}$、UniVLA 及真实机器人上验证了框架的跨骨干适用性和实际闭环控制价值
Card 04 方法描述

方法描述

  • 将标准VLA推理抽象为理解模块 $U_\theta$ 生成可复用语义条件 $z_t$，以及动作模块 $G_\phi$ 基于当前状态 $s_t$ 和语义条件生成动作 $A_t$
  • 理解模块异步低频刷新语义缓存（如最后一层隐藏状态或KV-cache），动作模块高频复用最新可用语义条件、当前状态及近期动作历史 $h_t$ 进行动作重规划
  • 训练时采用时间错位策略，使用过去时刻的过时语义条件 $z_{t-\Delta}$（$\Delta$ 在一个动作块内均匀采样）并结合带高斯噪声的历史动作，以模拟部署时的语义延迟和执行误差
Card 05 数据集与资源

数据集与资源

  • 仿真评估使用 LIBERO 基准（Spatial, Object, Goal, Long 四个任务集）
  • 真实机器人评估使用 SO100 和 Kinova Gen2 平台（共6个任务，每个任务10次试验）
  • 基础模型为 UniVLA 和 $\pi_{0.5}$
  • 训练资源：UniVLA 使用 8张 A100 GPU，全局批次大小128，训练30k步；$\pi_{0.5}$ 使用 2张 A100 GPU，批次大小64，训练15k步
Card 06 评估与结果

评估与结果

  • 评估环境为 LIBERO 仿真和真实机器人部署，控制回路频率固定为 20 Hz
  • 主要指标为任务成功率、加权完成时间和服务端推理吞吐量
  • UniVLA-Async 在真实机器人上的平均成功率达到 93.3%（优于同步 UniVLA 的 78.3%），SO100加权完成时间从17.4s缩短至 13.1s
  • 服务端推理吞吐量方面，UniVLA-Async 达到 35.6 Hz（对比同步版3.4 Hz），$\pi_{0.5}$-Async 达到 19.7 Hz
  • 消融实验证明：单纯的缓存复用会导致性能下降（LIBERO-Long 85.2 vs 同步 92.0），结合历史动作条件化和时间错位训练可恢复至 91.2
