Card 03 核心贡献

核心贡献

  • 提出 分层记忆管理框架，将机器人控制解耦为瞬时记忆（Executor）、工作记忆（Sentry）和情景记忆（Planner）三层，解决长期任务中的粒度冲突
  • 提出 跨模态记忆组织（融合视觉与文本语义）结合 主动管理机制（Add/Update/Delete），使机器人能在动态多模态交互中保持知识一致性与可塑性
  • 在复杂人机协作场景中实现 90% 平均任务成功率，显著降低 VLM 计算开销（API 调用减少约 3 倍），并展现出基于人类偏好的 自我修正能力
  • 实验证明分层架构能有效桥接与人类水平（Human High-level）的性能差距
Card 04 方法描述

方法描述

  • 三层策略分解：

- Executor (πₑ)：基于 π₀.₅ 的高频 VLA（2Hz 预测，10Hz 执行），仅依赖当前观测与激活的子目标，无记忆负担，确保物理稳定性

- Sentry (πₛ)：轻量级 VLM（Qwen3-VL-8B），维护滑动窗口工作记忆（𝒲ₜ，长度 hₛ=8），每 nₘ=5 步评估子目标完成状态，作为门控函数 uₜ 决定是否触发规划器

- Planner (πₚ)：重量级 VLM（GPT-4o 或 Qwen3-VL-30B），管理情景记忆（ℰₜ），仅在 Sentry 触发时（uₜ=1）进行上下文检索、记忆更新和重规划

  • 记忆架构：

- 瞬态记忆 (𝒯ₜ)：当前瞬时观测，用于反应式控制

- 工作记忆 (𝒲ₜ)：近期观测滑动窗口，用于子目标进度验证

- 情景记忆 (ℰₜ)：持久化长期存储，包含 情境记忆 ℰᶜ（多模态键值存储，支持 TopK 语义检索）和 过程记忆 ℰᵖ（语言子目标有序列表，含 active/pending/done 状态标签）

  • 动态管理机制：Planner 通过 Add（插入新事实）、Update（修正过时条目）、Delete（移除冗余/冲突信息）三种操作主动维护记忆，避免知识僵化
  • 推理流程：Executor 持续执行 → Sentry 周期性检测完成状态 → 触发时 Planner 检索相关上下文 → 更新记忆并生成新子目标 → 反馈给 Sentry 形成闭环
Card 05 数据集与资源

数据集与资源

  • 硬件平台：WidowX-250s 机械臂，配备平行夹爪与 Intel RealSense D435 第三视角相机 + 腕部相机
  • 训练数据：60 条真实机器人演示轨迹（50 条标准 + 10 条边界案例），通过遥操作收集，按 10Hz 下采样并 resize 至 224×224
  • 模型配置：

- Executor：基于 π₀.₅（DROID 数据集预训练）微调，使用 AdamW 优化器（lr=5e-5，batch size=256，30k 步）

- Sentry：Qwen3-VL-8B，部署于 1× H100 (80GB)，temperature=0.6，max length=8192

- Planner：GPT-4o（主实验）或 Qwen3-VL-30B（补充实验），部署于 2× H100 (80GB)，temperature=0.6，max length=16384

  • 记忆后端：向量数据库，使用 OpenAI text-embedding-3 模型编码文本查询，支持余弦相似度检索
Card 06 评估与结果

评估与结果

  • 评估任务：三个长期桌面操作任务（Object Search、Counting、Rearrangement），每个任务每个方法评估 20 次试验
  • 评估指标：

- Task Progress：完成的子目标比例，衡量任务完成质量

- API Calls：每个子任务平均 Planner 调用次数，衡量计算效率

- Memory Hit Rate：Planner 调用时所需信息存在于记忆中的概率，衡量记忆有效性

  • 主要结果：

- HiMe 平均任务进度达 90%，显著优于 Flat Memory（65%）、Transient Memory w/ Sentry（68%）等基线

- API 调用减少约 3 倍（如 Object Search：1.8 vs 5.4），系统延迟显著降低

- Memory Hit Rate 达 94%，远超 Flat Memory（68%），验证无限结构化记忆的有效性

- 在 Object Search 任务中，Sentry 使性能从 14% 提升至 26%，证明其通过减少任务切换确保执行一致性的核心价值

  • 消融实验：

- 跨模态记忆优于纯文本（91% vs 78% in Counting）或纯图像（86% vs 74% in Object Search）

- 主动管理必要：FIFO（68%）< 无管理（86%）< HiMe（90%），证明 Update/Delete 对消除冗余和冲突的关键作用

- Sentry 保守性：高精确率（~82%）但低召回率（~35%），倾向于继续执行而非过早停止，设计固定间隔回退机制防止无限循环
