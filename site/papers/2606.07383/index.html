Card 03 核心贡献

核心贡献

  • 识别出 VLM 的视觉和上下文 token 是 VLA 部署成本的主要瓶颈，并基于此选用 token 高效的 Qwen3-VL 骨干，在相近参数规模下实现比 π₀.₅ 更快的推理。
  • 提出 统一的跨机器人训练框架，包含 View Registry（视图注册表）、72 维统一物理状态-动作槽空间 和 机器人实例 LoRA，使异构机器人数据能在共享策略下训练。
  • 通过 算法-系统协同优化（硬件感知编译、混合精度执行、并行视觉编码），在 Huixi R1 边缘 SoC 上达到 11.69 Hz 的端到端推理性能，满足 10 Hz 闭环控制要求。
Card 04 方法描述

方法描述

  • 架构设计：采用 Qwen3-VL-2.13B 作为视觉-语言骨干，结合 0.40B 参数的 Action Expert，通过流匹配生成连续动作块。
  • 统一接口：View Registry 为每张图像附加相机角色与模态标签；72 维物理槽空间 为状态和动作赋予固定物理语义，并用二进制掩码标记有效维度；机器人实例 LoRA 插入 Action Expert 的 FFN 层，学习实例化残差行为。
  • 训练策略：在混合机器人数据集上进行联合预训练，冻结 VLM 骨干，只训练 VLM LoRA、共享 Action Expert 和实例 LoRA；采用 掩码流匹配损失 仅监督有效动作维度，并引入 残差正则化 防止实例 LoRA 过拟合。
  • 部署优化：针对 Huixi R1 芯片定制 FlashAttention 风格注意力核、算子融合 和 细粒度任务调度；采用 W8A16 混合精度 并设计融合解量化的 GEMM 内核；将三视角图像 批处理并行编码 以提升视觉编码效率。
Card 05 数据集与资源

数据集与资源

  • 训练数据：混合数据集，包含 Open X-Embodiment 的关节空间子集和 AgiBotWorld 的真实机器人轨迹，涵盖单臂、双臂、平行夹爪和灵巧手等多种平台。
  • 模型规模：VLM 骨干 2.13B 参数，Action Expert 0.40B 参数，总参数量约 2.53B。
  • 训练资源：未明确说明具体 GPU 类型，但提及使用 Huixi R1（7 nm 边缘 SoC，500 TOPS INT8 算力，200 GB/s 级内存带宽）进行部署。
Card 06 评估与结果

评估与结果

  • 评估基准：LIBERO 仿真基准（Spatial、Object、Goal、Long 四个子集）以及 AgiBot G1、G2 和 Galbot G1 三种真实机器人平台。
  • 主要指标：任务成功率（SR）、掩码流匹配损失、各动作维度（手臂、基座速度、偏航率、夹爪）的 MAE。
  • 关键结果：

- 在 LIBERO 基准上平均成功率 90.0%，其中 Long 子集达 82.4%，超过多数直接策略和 VLA 基线。

- 真实机器人任务中，在 seen 和 unseen 设置下均取得高成功率（具体数值见原文 Table 5）。

- 端到端推理在 Huixi R1 上达到 11.69 Hz，满足 10 Hz 实时控制目标。
