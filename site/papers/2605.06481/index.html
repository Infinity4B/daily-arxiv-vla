OA-WAM: Object-Addressable World Action Model for Robust Robot Manipulation

2026-05-07 · 原文 · 翻译 · 2605.06481

论文提出 OA-WAM，一种对象可寻址的世界动作模型，旨在解决现有世界动作模型在场景扰动下缺乏对象可寻址性导致的鲁棒性不足问题。 核心创新是将每帧场景分解为 N+1 个对象槽，每个槽分为冻结的身份地址和时变的内容向量，并通过架构约束确保动作解码器能稳定寻址任务相关对象。 论文目标是实现更鲁棒的机器人操作，特别是在对象布局、相机视角、机器人初始状态等几何扰动下保持高性能。

Card 03 核心贡献

核心贡献

  • 将现有世界动作模型在扰动下表现不佳的问题根源诊断为缺乏对象可寻址性，即世界状态未能提供稳定的、可查询的对象级接口。
  • 提出 OA-WAM 模型，包含对象槽状态 token、块因果世界-动作序列、对象可寻址注意力约束（仅地址键投影 + 每层地址重置）等关键设计。
  • 在标准基准和鲁棒性基准上验证设计有效性，匹配 SOTA 性能并在与假设一致的几何扰动轴上领先，消融实验证实对象可寻址接口是鲁棒性的关键来源。
Card 04 方法描述

方法描述

  • 使用冻结的基础感知模型（SAM 3 + DINOv3 + Qwen3-VL）生成六路 token 流（文本、图像 VQ、对象槽、本体感受、历史动作），融合为块因果序列。
  • 每个对象槽向量由 32 维冻结地址、256 维时变内容、帧索引嵌入和角色嵌入组成，通过可学习槽适配器映射到 trunk 隐藏维度。
  • 核心创新是对象可寻址注意力：在注意力键投影中仅使用地址子向量，并在每层 transformer 后用前向钩子将残差流的地址切片重置为缓存身份，架构级分离对象身份寻址与时变内容建模。
Card 05 数据集与资源

数据集与资源

  • 使用 LIBERO、SimplerEnv 和 LIBERO-Plus 基准进行评估。
  • 模型基于 7B Chameleon 多模态 trunk，包含约 127M 可训练参数（80M LoRA + 47M 预测头）。
  • 训练采用三阶段流程，但原文未明确提及具体 GPU/TPU 训练资源。
Card 06 评估与结果

评估与结果

  • 评估环境为三个仿真基准，使用成功率作为主要指标。
  • 在 LIBERO 上达到 97.8% 平均成功率，在 SimplerEnv 上达到 79.3%，领先所有 VLA 和 WAM 基线。
  • 在 LIBERO-Plus 鲁棒性基准上，在相机扰动轴达到 80.5%，机器人初始状态轴达到 89.6%，几何轴平均达到 84.3%，均创 SOTA。
  • 因果对象干预测试中，OA-WAM 的交换绑定余弦达到 0.87，远高于所有整体基线（≤0.09）。
