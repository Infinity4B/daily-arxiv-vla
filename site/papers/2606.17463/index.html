WeaveLA: Event Driven Cross-Subtask Latent Memory Weaving for Repetitive Robot Manipulation

Card 02 论文概述

论文概述

  • 论文针对重复性机器人操作任务中当前阶段依赖前一阶段完成状态的问题，提出了WeaveLA跨子任务记忆接口。
  • 指出现有短窗口VLA策略缺乏跨子任务边界传递信息的显式通道，导致在需要跨子任务依赖的操作中表现脆弱。
  • 核心观点是子目标完成事件是跨子任务记忆传递的自然时间单元，并在该时刻将压缩的潜在记忆直接路由到下一子任务的动作生成路径。
Card 03 核心贡献

核心贡献

  • 识别子目标完成事件作为跨子任务记忆传递的自然时间单元，强调了记忆写入“何时”触发的设计维度。
  • 提出WeaveLA架构，通过查询驱动的注意力池化机制在每个子目标边界写入8个潜在令牌，并直接注入下一个子任务的动作专家，主干网络保持冻结。
  • 通过重复分层、难度分层和逐回合配对分析，提供了机制对齐的证据：增益仅在需要跨子任务通道的任务上体现，而在非重复任务上保持中性。
Card 04 方法描述

方法描述

  • 采用事件驱动的触发机制，仅在子目标完成事件发生时触发记忆写入，而非逐帧写入。
  • 查询驱动的记忆编织器利用8个可学习查询令牌，通过单步注意力池化将刚完成的片段压缩为紧凑的潜在表示。
  • 动作侧记忆消费：将编织器产生的记忆令牌通过AdaRMS调制直接注入Gemma动作专家的动作生成路径中，保持视觉和语言上下文不变。
  • 采用分阶段训练策略（Stage 0: 任务集适应，Stage 1: Weaver预热，Stage 2: 加入语义对齐和对比辅助损失）以防止潜在表示坍缩。
  • 提出一种模型内部的潜在偏移检测器作为非Oracle触发源，在部署时无需外部监督即可检测子目标边界。
Card 05 数据集与资源

数据集与资源

  • 使用RoboMME基准测试集（包含16个操作任务，重点评估SwingXtimes、StopCube、PickXtimes）。
  • 基于冻结的$\pi_{0.5}$（包含PaliGemma和Gemma组件）骨干网络进行实例化。
  • 可训练参数量约为46M，仅占基础策略（约3.4B）的1.4%。
  • 推理时记忆通道的运行时间开销因事件驱动的特性而可忽略不计。
Card 06 评估与结果

评估与结果

  • 评估环境为RoboMME仿真环境及真实机器人物理环境。
  • 主要评估指标为逐任务成功率（50个回合）。
  • 在6任务聚合评估中，成功率从19.0%提升至24.7%；在16任务评估中从17.3%提升至23.3%。
  • 在最难的重复切片SwingXtimes N=3中，成功率从0%跃升至47.8%（6任务），在16任务中从4%提升至30%。
  • 在无需跨子任务依赖的任务（N=1）中，成功率保持在约100%，验证了机制的精准激活而不引入 incidental capacity。
  • 真实机器人PickXtimes（N=3）测试中，Weaver-off为45%，Weaver-on提升至70%。
  • 使用潜在偏移检测器作为触发源（非Oracle）可恢复SwingXtimes上68%的Oracle性能差距。
