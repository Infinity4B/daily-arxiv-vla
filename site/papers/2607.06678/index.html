Card 03 核心贡献

核心贡献

  • 提出Native Memory Compression，将每帧-视图压缩为单个原生记忆token（M=1），实现极致的token效率
  • 设计两阶段训练pipeline：第一阶段在冻结预训练VLA上用原生动作损失学习通用记忆分词器，第二阶段针对任务微调完整VLA
  • 无需外部规划器或新初始化记忆模块，保持与预训练VLA架构完全兼容，可直接利用其原有注意力机制
  • 在模拟和真实机器人任务上均达到SOTA，同时保持低推理延迟和GPU内存占用（32GB预算下支持5000帧）
  • 数据效率高，仅用20%训练数据即可达到 prior arts 水平，真实机器人仅需100个演示即可转换预训练策略
Card 04 方法描述

方法描述

  • 从预训练VLA视觉编码器初始化记忆编码器 $E_{mem}$ 和可学习查询token $q_{mem}$
  • 通过注意力机制将视觉patch序列 $P_\tau^v$ 聚合为单个记忆摘要token $\hat{q}_\tau^v$
  • 线性投影 $W_{mem}$ 将摘要映射到VLA token维度，生成原生记忆token $m_\tau^v$
  • 记忆序列 $[b_{mem}, \{m_\tau^v\}]$ 追加到原始VLA输入序列 $[x^{obs}, x^{prompt}, M_t]$
  • 第一阶段：冻结VLA骨干，仅训练记忆分支（记忆编码器、查询、投影、BOS token），用VLA原生动作损失优化
  • 第二阶段：冻结记忆分词器，微调VLA骨干、动作头和记忆投影
  • 推理时记忆分词器独立运行，与VLA策略解耦，可高频更新历史记忆
Card 05 数据集与资源

数据集与资源

  • 模拟数据集：RMBench任务（3个）、Click Buttons、Click Buttons (Hard)、Swap Blocks、Put Back Block、Observe and Pickup
  • 真实机器人任务：Click Buttons、Put Back Block、Grocery Checkout Scanning（unseen，未参与分词器训练）
  • 模型架构：基于pi_0.5的VLA骨干
  • 训练资源：混合仿真和真实演示数据，包含标准操作和记忆密集型任务；第一阶段记忆分词器训练，第二阶段任务微调
  • 推理资源：32GB GPU内存预算下支持5000帧历史，100ms实时延迟约束下可 attends 约200帧
Card 06 评估与结果

评估与结果

  • 模拟平均成功率84.0%，较Mem-0的32.4%提升显著，在5个任务上均达到最佳
  • 真实机器人平均成功率98.7%，较pi_0.5 + RTC的34.7%大幅提升，在Grocery Checkout Scanning unseen任务上表现稳定
  • 推理效率：支持分钟级历史（160+帧），比 prior methods 长9-40倍；记忆更新与策略推理解耦
  • 数据效率：仅用100个真实任务演示（约5小时）即可转换预训练单帧VLA；仅用10个演示即达60%成功率
  • 消融实验表明原生记忆压缩对保留任务相关细节至关重要，密集时间覆盖和长期记忆对性能均有贡献
