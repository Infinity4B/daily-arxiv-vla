Safety Case Patterns for VLA-based driving systems: Insights from SimLingo

2026-03-16 · 原文链接 · 幻觉翻译 · 2603.16013

研究单位

  • York University（加拿大约克大学）
  • National Institute of Informatics（日本国立情报学研究所）

论文概述

  • 论文关注 Vision–Language–Action (VLA) 驾驶系统的安全性问题，特别是用户指令引入的潜在风险。
  • 指出将自然语言输入（如导航指令）集成到控制回路可能导致不可预测的危险行为，现有保障方法无法有效应对此类基于指令的新型风险。
  • 提出了一种名为 RAISE 的安全案例设计方法，旨在为 VLA 驾驶系统构建严谨、基于证据的安全声明。

核心贡献

  • 提出了 HARA（危险分析与风险评估） 的扩展方法，用于分析用户指令可能导致的危险动作及安全指令的结果。
  • 提出了两种新颖的安全案例模式：RI（Reject Instruction）模式 和 AAI（Accept Adequate Instructions）模式，用于论证系统拒绝危险指令和接受安全指令的能力。
  • 提出了 RAISE 方法，利用扩展的 HARA 结果和提出的模式构建具体的 VLA 驾驶系统安全案例模型。
  • 通过 SimLingo 系统的案例研究验证了所提方法的适用性。

方法描述

  • 采用 Goal Structuring Notation (GSN) 图形化符号表示安全案例。
  • 扩展了传统的 HARA 流程，增加了 Safe Events（安全事件） 识别环节，以涵盖安全指令的场景结果。
  • 设计了包含“热点”的模式库，工程师可将其替换为具体系统信息以实例化安全案例。
  • 提出了一种算法，通过迭代选择模式并分解安全目标，将高层安全目标逐步细化为具体的子目标和证据。

数据集与资源

  • 使用了 SimLingo 作为主要的参考 VLA 驾驶系统进行案例研究。
  • 利用 CARLA 自动驾驶模拟器作为仿真环境。
  • 参考了 LMDrive 系统的场景信息以辅助模式的泛化分析。
  • 案例研究结果及算法已发布在 GitHub 仓库中。

评估与结果

  • 评估环境为 CARLA 模拟器，假设车辆处于行动模式且传感器为相机。
  • 评估指标集中在系统功能的分解与论证，如系统能否正确处理相机信息、到达目的地、拒绝危险指令等。
  • 实验结果表明，所提方法能够成功构建 SimLingo 的顶层安全案例，并有效论证其在特定场景下拒绝危险指令和接受安全指令的安全性。
