AdaThinkDrive: Adaptive Thinking via Reinforcement Learning for Autonomous Driving

2025-09-17 · 原文链接 · 幻觉翻译 · 2509.13769

论文研究单位

清华大学（Tsinghua University）、小米EV（Xiaomi EV）、澳门大学（University of Macau）、南洋理工大学（Nanyang Technological University）、北京大学（Peking University）

论文概述

AdaThinkDrive提出一种“快速回答/慢速思考”的双模推理机制，解决在端到端自动驾驶中思维链（CoT）在简单场景下过度推理、复杂场景下推理收益不一致的问题。论文在Navsim基准上进行实验与评估。

论文核心贡献点

  • 通过在Navsim上系统性对比发现：简单场景下使用CoT收益有限甚至有害，复杂场景下CoT收益显著。由此提出“按场景复杂度自适应推理”的必要性。
  • 设计AdaThinkDrive框架：三阶段训练范式——预训练获得驾驶常识与知识、两阶段SFT学习双模输出能力、基于GRPO的强化学习通过奖励信号学习“何时思考与何时直接作答”。
  • 提出“Adaptive Think Reward”策略：通过多 rollout 采样，比较同一场景下Thinking与Non-thinking轨迹质量（PDMS），动态奖惩并更新场景复杂度标签，避免对固定人工标签的依赖。
  • 在Navsim达成SOTA：PDMS 90.3（仅视觉输入），比最佳纯视觉基线提升1.7；同时在复杂场景中优先使用思考模式（96%），在简单场景中优先直接输出（84%）；相对Always-Think推理时间降低14%。

论文方法描述

  • 问题建模：以联合分布P(m, o \|q)建模对查询q选择模式m（Thinking/Non-thinking）与输出轨迹o的决策；通过最大化期望任务效用U(q, o)选择最优模式。
  • 数据准备：

- 预训练数据：DriveLM、LingoQA、ImpromptuVLA、NuScenes-QA、NuInstruct、OmniDrive等开源驾驶QA数据，用于获得基础驾驶常识。

- 混合SFT数据：为同一查询生成Thinking（含完整推理轨迹）与Non-thinking（省略推理仅含轨迹）两种输出；场景注释通过Qwen2.5-VL-72B自动生成，包含交通灯状态、天气与道路边界等；动态交互对象分为CIPO-1（本车道障碍物）、CIPO-2（可能并线）、Motion Interaction（未来轨迹与自车相交）。

- 场景分类：将Navsim训练与验证集按“是否靠近道路边界/是否有关键物体”划分为Level 1（简单，无条件）、Level 2（有单一条件）、Level 3（复杂，同时满足两个条件），并映射为D+（Level 2&3）与D-（Level 1）作为强化学习初始辅助标签。

  • 两阶段SFT：

- 阶段一：在大规模驾驶QA数据上微调，提升场景理解与驾驶语义认知能力。

- 阶段二：在Navsim规划数据（含Thinking/Non-thinking双模输出）上微调，使模型具备统一接口生成两种输出风格且不偏向其一。

  • 自适应思考强化学习（基于GRPO）：

- 奖励设计：

- PDMS Reward：基于轨迹质量评价PDMS。

- Format Reward：强制输出遵循、标签与轨迹格式规范。

- Endpoint Reward：对轨迹终点误差进行分段奖励（<2m=1.0；<4m=0.8；<6m=0.6；<10m=0.4；<15m=0.2；否则=0）。

- Adaptive Think Reward：依据多rollout比较Thinking与Non-thinking的平均PDMS与样本数，在D=0/D=1的当前标签下动态奖惩；当满足阈值T与条件时允许修正场景复杂度标签为“challenging/simple”。

- 优化目标：采用GRPO优化，引入截断重要性权重与KL正则防止策略偏移。

论文使用数据集和训练资源

  • 数据集：Navsim（主评估）、DriveLM、LingoQA、ImpromptuVLA、NuScenes-QA、NuInstruct、OmniDrive（用于预训练与知识注入）
  • 模型：InternVL3-8B作为基础视觉-语言模型
  • 训练资源：

- 阶段一：学习率1e-5，批量1，训练2个epoch

- 阶段二：学习率4e-5，批量2，训练2个epoch

- 阶段三（RL）：学习率2e-6，批量4，使用64×NVIDIA H20 GPU进行2个epoch

论文使用的评估环境和评估指标

  • 评估环境：Navsim非反应式仿真，OpenScene平台支持
  • 评估指标：

- PDMS（Predictive Driver Model Score）：综合评分，融合NC、DAC、TTC、CF、EP

- 推理时间：预测4秒轨迹的平均推理时间（Navsim测试集）

- 场景分层：按Level 1/2/3的Think vs Non-think选择比例与性能表现

  • 定量结果（主要指标）：

- AdaThinkDrive（仅视觉）：PDMS 90.3，NC 98.4、DAC 97.8、TTC 95.2、CF 100、EP 84.4

- Best-of-N：PDMS 93.0

- 相对Non-Think RL：PDMS +2.0；推理时间 Non-Think RL 0.68s、Think RL 0.86s、AdaThinkDrive 0.74s

- 相对Always-Think：推理时间减少14%
